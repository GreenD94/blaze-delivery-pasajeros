Flujo de Datos del Pasajero

1. Registro y Autenticación

El pasajero se registra y autentica en el sistema.

2. Solicitud de Viaje

El pasajero solicita un viaje especificando origen y destino.

3. Asignación de Conductor

El sistema asigna un conductor disponible al viaje.

4. Ejecución del Viaje

El pasajero realiza el viaje con seguimiento en tiempo real.

5. Pago y Calificación

Se procesa el pago y el pasajero califica la experiencia.

📱 Registro y Autenticación

Proceso de Registro del Pasajero

1. Registro de Usuario

El pasajero crea una cuenta con información personal básica.

Endpoint: POST /api/register

Registro de nuevo usuario pasajero

// Request de registro { "first_name": "María", "last_name": "González", "username": "maria_gonzalez", "email": "maria@email.com", "user_type": "rider", "contact_number": "+1234567890", "password": "password123", "player_id": "onesignal_player_id" }

2. Verificación de Email

El sistema envía un email de verificación al usuario.

📧 Verificación Requerida

El usuario debe verificar su email antes de poder solicitar viajes.

3. Inicio de Sesión

El pasajero inicia sesión con sus credenciales.

Endpoint: POST /api/login

Autenticación del pasajero

// Request de login { "email": "maria@email.com", "password": "password123", "user_type": "rider" }

4. Configuración de Perfil

El pasajero completa su perfil con información adicional.

// Estructura del perfil del pasajero { "id": 123, "first_name": "María", "last_name": "González", "email": "maria@email.com", "contact_number": "+1234567890", "profile_image": "https://example.com/avatar.jpg", "emergency_contacts": [ { "name": "Juan Pérez", "phone": "+1234567891", "relationship": "Esposo" } ], "preferences": { "payment_method": "wallet", "language": "es", "notifications": true } }

🔐 Métodos de Autenticación

  • Email/Contraseña: Método tradicional
  • OTP por SMS: Verificación por código
  • Google Sign-In: Integración con Google
  • Facebook Login: Integración con Facebook
  • Apple Sign-In: Para dispositivos iOS

📋 Información Requerida

  • Nombre completo: Nombre y apellidos
  • Email: Dirección de correo válida
  • Teléfono: Número de contacto
  • Contraseña: Mínimo 8 caracteres
  • Foto de perfil: Opcional

🗺️ Solicitud y Reserva de Viaje

Proceso de Solicitud de Viaje

1. Selección de Origen y Destino

El pasajero selecciona puntos de origen y destino en el mapa.

// Ejemplo de selección de ubicación LocationModel origin = LocationModel( latitude: 40.7128, longitude: -74.0060, address: "Nueva York, NY" ); LocationModel destination = LocationModel( latitude: 40.7589, longitude: -73.9851, address: "Times Square, NY" );

2. Estimación de Precio

El sistema calcula el precio estimado del viaje.

Endpoint: POST /api/estimate-price-time

Calcula el precio estimado del viaje

// Request de estimación { "start_latitude": "40.7128", "start_longitude": "-74.0060", "end_latitude": "40.7589", "end_longitude": "-73.9851", "service_id": 1 }

3. Selección de Tipo de Servicio

El pasajero elige entre diferentes tipos de vehículos.

🚗 Económico

Vehículos compactos, tarifa más baja

🚙 Estándar

Vehículos medianos, tarifa regular

🚘 Premium

Vehículos de lujo, tarifa elevada

🚐 XL

Vehículos grandes, más pasajeros

4. Confirmación de Solicitud

El pasajero confirma y envía la solicitud al sistema.

Endpoint: POST /api/save-riderequest

Crea nueva solicitud de viaje

// Request de solicitud { "rider_id": 123, "service_id": 1, "datetime": "2024-01-15T10:00:00Z", "start_latitude": "40.7128", "start_longitude": "-74.0060", "start_address": "Nueva York, NY", "end_latitude": "40.7589", "end_longitude": "-73.9851", "end_address": "Times Square, NY", "seat_count": 1, "status": "new_ride_requested", "payment_type": "wallet", "proposed_fee": 25.50 }

💰 Componentes de la Tarifa

  • Tarifa base: Cargo inicial
  • Por distancia: Cargo por km recorrido
  • Por tiempo: Cargo por minuto de viaje
  • Cargos adicionales: Peajes, espera
  • Descuentos: Cupones aplicados

⏱️ Tiempos de Respuesta

  • Estimación de precio: 2-3 segundos
  • Búsqueda de conductor: 30-60 segundos
  • Asignación de conductor: 15-30 segundos
  • Confirmación de viaje: Inmediata

⏱️ Asignación y Seguimiento del Conductor

Proceso de Asignación de Conductor

1. Búsqueda de Conductores Cercanos

El sistema busca conductores disponibles en un radio de 5km del origen.

Endpoint: GET /api/nearby-drivers

Obtiene lista de conductores cercanos

// Respuesta de conductores cercanos { "success": true, "drivers": [ { "id": 456, "name": "Carlos López", "rating": 4.8, "vehicle": { "model": "Toyota Camry", "plate": "ABC123", "color": "Blanco" }, "distance": 1.2, "estimated_arrival": "5 minutos" } ] }

2. Notificación MQTT

El pasajero recibe notificaciones en tiempo real sobre el estado del viaje.

// Ejemplo de notificación MQTT { "success_type": "accepted", "result": { "id": 12345, "driver_name": "Carlos López", "driver_phone": "+1234567890", "vehicle_info": "Toyota Camry - ABC123", "estimated_arrival": "5 minutos", "driver_rating": 4.8 } }

3. Seguimiento en Tiempo Real

El pasajero puede ver la ubicación del conductor en tiempo real.

Endpoint: GET /api/current-riderequest

Obtiene información actual del viaje

4. Estados del Viaje

El sistema actualiza el estado del viaje en tiempo real.

🔄 Estados del Viaje

PENDIENTE
ACEPTADO
LLEGÓ
INICIADO
COMPLETADO

📍 Características de Seguimiento

  • Actualización GPS: Cada 12 segundos
  • Precisión: Alta precisión
  • Ruta optimizada: Tiempo más corto
  • Tráfico en tiempo real: Sí
  • Tiempo estimado: Actualizado constantemente

📱 Notificaciones Push

  • Conductor asignado: Inmediata
  • Conductor llegando: 2 minutos antes
  • Conductor llegó: Al llegar
  • Viaje iniciado: Al comenzar
  • Viaje completado: Al finalizar

👤 Verificación y Recogida del Conductor

Proceso de Verificación del Conductor

1. Información del Conductor

El pasajero recibe información detallada del conductor asignado.

// Información del conductor { "driver": { "id": 456, "first_name": "Carlos", "last_name": "López", "profile_image": "https://example.com/driver.jpg", "rating": 4.8, "total_rides": 1250, "vehicle": { "model": "Toyota Camry", "plate": "ABC123", "color": "Blanco", "year": 2020 }, "contact_number": "+1234567890" } }

2. Verificación OTP

El conductor debe ingresar un código OTP para confirmar la recogida.

🔐 Código OTP

El sistema genera un código de 4 dígitos que el conductor debe ingresar para confirmar la recogida del pasajero.

3. Confirmación de Recogida

El conductor confirma que el pasajero está en el vehículo.

Endpoint: POST /api/riderequest-update/{rideId}

Actualiza el estado del viaje a "IN_PROGRESS"

// Request de inicio de viaje { "id": 12345, "status": "in_progress" }

4. Inicio del Viaje

El viaje comienza oficialmente y se inicia el seguimiento.

IN_PROGRESS

🔒 Medidas de Seguridad

  • Verificación OTP: Obligatoria
  • Foto del conductor: Disponible
  • Información del vehículo: Completa
  • Calificación del conductor: Visible
  • Historial de viajes: Accesible

📞 Comunicación

  • Chat en tiempo real: Disponible
  • Llamadas directas: Número enmascarado
  • Mensajes predefinidos: Respuestas rápidas
  • Notificaciones: Push automáticas
  • Traducción: Múltiples idiomas

🚗 Experiencia de Viaje y Navegación

Proceso Durante el Viaje

1. Seguimiento en Tiempo Real

El pasajero puede ver la ubicación exacta del vehículo en el mapa.

// Actualización de ubicación del conductor { "ride_id": 12345, "driver_location": { "latitude": 40.7128, "longitude": -74.0060, "timestamp": "2024-01-15T10:30:00Z", "speed": 35, "heading": 90 }, "estimated_arrival": "2024-01-15T10:45:00Z" }

2. Comunicación con el Conductor

El pasajero puede comunicarse con el conductor mediante chat.

WebSocket: /chat/{rideId}

Conexión WebSocket para chat en tiempo real

// Mensaje de chat { "ride_id": 12345, "sender_id": 123, "sender_type": "rider", "message": "¿En qué puerta me encuentras?", "timestamp": "2024-01-15T10:30:00Z" }

3. Gestión de Emergencias

El pasajero tiene acceso a botones de emergencia durante el viaje.

🚨 Botón SOS

El pasajero puede activar el botón SOS en caso de emergencia, lo que notifica inmediatamente a los contactos de emergencia y al soporte.

4. Llegada al Destino

El conductor confirma la llegada al destino final.

COMPLETED

  • Puntos de interés: Cercanos al destino
  • Modo offline: Disponible

💬 Funciones de Comunicación

  • Chat en tiempo real: WebSocket
  • Llamadas directas: Integración nativa

💳 Pago y Gestión de Tarifas

Proceso de Pago

1. Cálculo de Tarifa Final

Al finalizar el viaje, se calcula la tarifa basada en distancia real y tiempo.

// Cálculo de tarifa final { "ride_id": 12345, "base_fare": 20.00, "distance_charge": 5.20, "time_charge": 3.75, "extra_charges": 2.50, "coupon_discount": 0.00, "total_amount": 31.45, "currency": "USD" }

2. Selección de Método de Pago

El pasajero elige entre múltiples métodos de pago disponibles.

👛 Wallet

Saldo de la aplicación

💳 Tarjeta

Crédito/Débito

💰 Efectivo

Pago directo

📱 Pago Móvil

Transferencia bancaria

3. Procesamiento del Pago

Se procesa el pago según el método seleccionado.

Endpoint: POST /api/save-payment

Procesa el pago del viaje

// Request de pago { "id": 12345, "rider_id": 123, "ride_request_id": 12345, "datetime": "2024-01-15T10:45:00Z", "total_amount": 31.45, "payment_type": "wallet", "txn_id": "", "payment_status": "paid", "transaction_detail": "" }

4. Confirmación de Pago

El sistema confirma el pago exitoso y genera el recibo.

✅ Pago Exitoso

El pago se ha procesado correctamente. Se ha enviado un recibo por email.

💰 Métodos de Pago Disponibles

  • Wallet interno: Saldo de la aplicación
  • Tarjeta de crédito/débito: Visa, Mastercard
  • Efectivo: Pago directo al conductor
  • Pago móvil: Transferencia bancaria
  • PayPal: Integración con PayPal

📊 Métricas de Pago

  • Tasa de éxito: 99.2%
  • Tiempo de procesamiento: 2 segundos
  • Método preferido: Wallet (60%)
  • Disputas: 0.1%
  • Reembolsos: Automáticos

⭐ Sistema de Calificación y Comentarios

Proceso de Calificación

1. Solicitud de Calificación

Al completar el viaje, se solicita al pasajero calificar la experiencia.

// Solicitud de calificación { "ride_id": 12345, "driver_name": "Carlos López", "driver_rating": 4.8, "request_rating": true }

2. Calificación del Conductor

El pasajero califica al conductor de 1 a 5 estrellas.

Endpoint: POST /api/save-ride-rating

Envía la calificación del viaje

// Request de calificación { "ride_request_id": 12345, "rating": 5, "comment": "Excelente servicio, muy puntual y amable", "tips": 5.00 }

3. Comentarios Opcionales

El pasajero puede agregar comentarios detallados sobre la experiencia.

📝 Comentarios

Los comentarios ayudan a mejorar el servicio y proporcionan retroalimentación valiosa al conductor.

4. Propinas

El pasajero puede agregar propinas al conductor.

// Opciones de propina { "tip_options": ["5.00", "10.00", "15.00", "20.00"], "custom_tip": true, "tip_amount": 10.00 }

⭐ Criterios de Calificación

  • Puntualidad: Llegada a tiempo
  • Cortesía: Trato amable
  • Limpieza: Vehículo limpio
  • Conducción: Segura y suave
  • Comunicación: Clara y efectiva

💰 Sistema de Propinas

  • Propinas sugeridas: $5, $10, $15, $20
  • Propina personalizada: Cantidad libre
  • Propina promedio: $8.50
  • Propina automática: Opcional
  • Propina por porcentaje: 10%, 15%, 20%

🚨 Características de Seguridad y Emergencia

Sistema de Seguridad del Pasajero

1. Botón SOS

El pasajero puede activar el botón SOS en caso de emergencia.

Endpoint: POST /api/admin-sos-notify

Notifica emergencias al administrador

// Request de emergencia { "ride_request_id": 12345, "latitude": 40.7128, "longitude": -74.0060, "emergency_type": "safety_concern", "description": "Situación de emergencia" }

2. Contactos de Emergencia

El pasajero puede configurar contactos de emergencia.

// Contactos de emergencia { "emergency_contacts": [ { "name": "Juan Pérez", "phone": "+1234567891", "relationship": "Esposo", "notify_on_sos": true }, { "name": "María García", "phone": "+1234567892", "relationship": "Hermana", "notify_on_sos": true } ] }

3. Compartir Viaje

El pasajero puede compartir su ubicación en tiempo real con contactos.

📍 Compartir Ubicación

El pasajero puede compartir su ubicación en tiempo real con familiares o amigos durante el viaje.

4. Verificación de Conductor

El sistema verifica la identidad del conductor antes del viaje.

⚠️ Verificación Obligatoria

El conductor debe verificar su identidad mediante OTP antes de poder recoger al pasajero.

🆘 Funciones de Emergencia

  • Botón SOS: Acceso rápido
  • Notificación automática: A contactos de emergencia
  • Ubicación en tiempo real: GPS activo
  • Llamada de emergencia: 911 automática
  • Respuesta inmediata: 24/7

🔒 Medidas de Seguridad

  • Verificación de identidad: OTP obligatorio
  • Tracking continuo: Ubicación registrada
