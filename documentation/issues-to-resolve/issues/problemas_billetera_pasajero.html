💸 El saldo de la billetera no se muestra en la interfaz de pasajero

📝 Por Hacer
⬅️ Volver a la lista de problemas | 📋 Estándares de Documentación

📍 Ubicación del problema

  • Archivo: lib/model/WalletListModel.dart y pantalla de billetera de pasajero
  • Función: Deserialización del modelo y renderizado del saldo

📝 Descripción detallada

El saldo de la billetera no se muestra en la interfaz de pasajero, aunque la API responde correctamente con los datos. En los logs se observa el error: type '_Map<String, dynamic>' is not a subtype of type 'String?'. Esto indica que el modelo espera un String? pero la API puede enviar un objeto Map, causando un fallo en la deserialización y evitando que el saldo se muestre.

💡 Solución recomendada

  1. Revisar el modelo WalletListModel y cambiar los campos que pueden ser objeto a dynamic o Map<String, dynamic>? en vez de String?.
  2. Ajustar el método fromJson para manejar ambos tipos de datos.
  3. Verificar en la UI que el campo wallet_balance se muestre correctamente y que no haya errores de tipo.
// Ejemplo de ajuste en el modelo
final dynamic data; // o Map? data;

// En fromJson
// data: json['data'] is String || json['data'] == null ? json['data'] : json['data'] as Map,

🚀 Ventajas de la solución

  • El usuario podrá ver correctamente su saldo de billetera.
  • Se evitan errores de tipo y posibles crashes en la app.
  • Mejora la experiencia y confianza del usuario.

📅 Fecha y autor
