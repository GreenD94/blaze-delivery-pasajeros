🚗 Nueva Vista - Últimas 30 Carreras del Conductor

Objetivo: Crear una vista administrativa que muestre las últimas 30 carreras de un conductor con detalles completos de pagos, efectivo recibido, vuelto y saldos de billetera.

📊 Análisis de Viabilidad

✅ RESULTADO: Es completamente posible crear esta vista con las tablas existentes en la base de datos.

🔍 Tablas Principales Identificadas:

  • ride_requests - Información principal de las carreras
  • payments - Detalles de pagos y transacciones
  • users - Datos de conductores y pasajeros
  • wallet_histories - Historial de movimientos de billetera
  • wallets - Saldos actuales de billetera

🎯 Campos Clave Disponibles

En tabla `ride_requests`:

Campo Descripción Uso
id ID único de la carrera Identificación
driver_id ID del conductor Filtro por conductor

En tabla `payments`:

Campo Descripción Uso
ride_request_id ID de la carrera Relación con carrera
total_amount Monto del pago Verificación de monto
payment_type Tipo de pago Método utilizado
collected_cash Efectivo cobrado Monto en efectivo
txn_id ID de transacción Referencia de pago
payment_status Estado del pago Confirmación

💻 Query SQL Principal

SELECT rr.id AS ride_id, rr.datetime, rr.total_amount, rr.payment_type, rr.cash_in_hand, rr.cash_collected, -- Información del conductor CONCAT(d.first_name, ' ', d.last_name) AS driver_name, d.contact_number AS driver_phone, -- Información del pasajero CONCAT(r.first_name, ' ', r.last_name) AS rider_name, r.contact_number AS rider_phone, -- Detalles de pago p.total_amount AS payment_amount, p.payment_type AS payment_method, p.collected_cash, p.txn_id, p.payment_status, -- Información de billetera w.total_amount AS rider_wallet_balance, -- Cálculos de vuelto CASE WHEN rr.payment_type = 'cash' THEN (rr.cash_in_hand - rr.total_amount) ELSE 0 END AS change_given, -- Estado de la carrera rr.status AS ride_status FROM ride_requests rr LEFT JOIN users d ON rr.driver_id = d.id LEFT JOIN users r ON rr.rider_id = r.id LEFT JOIN payments p ON rr.id = p.ride_request_id LEFT JOIN wallets w ON r.id = w.user_id WHERE rr.driver_id = ? -- ID del conductor AND rr.status IN ('completed', 'finished') ORDER BY rr.datetime DESC LIMIT 30;

Información que se puede mostrar

✅ Datos Disponibles:

  • ID y fecha de la carrera
  • Monto total cobrado
  • Tipo de pago (efectivo, billetera, móvil)
  • Efectivo recibido (cash_in_hand)
  • Efectivo cobrado (cash_collected)
  • Vuelto calculado (efectivo recibido - monto total)
  • Saldo de billetera del pasajero
  • Datos del conductor y pasajero
  • Estado del pago

⚠️ Limitaciones Identificadas:

  • No hay campo específico para "billete recibido" (solo monto total)
  • No hay campo para "saldo deducido de billetera" (solo saldo actual)
  • El vuelto se calcula, no se almacena directamente

🔧 Recomendaciones de Mejora

1. Agregar campos adicionales a la tabla ride_requests:

ALTER TABLE ride_requests ADD COLUMN bill_received DECIMAL(10,2) DEFAULT 0; ALTER TABLE ride_requests ADD COLUMN change_amount DECIMAL(10,2) DEFAULT 0; ALTER TABLE ride_requests ADD COLUMN wallet_deduction DECIMAL(10,2) DEFAULT 0;

2. Crear una vista para facilitar consultas:

CREATE VIEW driver_ride_summary AS SELECT rr.id, rr.datetime, rr.total_amount, rr.payment_type, rr.cash_in_hand, rr.cash_collected, (rr.cash_in_hand - rr.total_amount) AS change_given, CONCAT(d.first_name, ' ', d.last_name) AS driver_name, CONCAT(r.first_name, ' ', r.last_name) AS rider_name, p.payment_status, w.total_amount AS rider_wallet_balance FROM ride_requests rr LEFT JOIN users d ON rr.driver_id = d.id LEFT JOIN users r ON rr.rider_id = r.id LEFT JOIN payments p ON rr.id = p.ride_request_id LEFT JOIN wallets w ON r.id = w.user_id WHERE rr.status IN ('completed', 'finished');

3. Query simplificado usando la vista:

SELECT * FROM driver_ride_summary WHERE driver_id = ? ORDER BY datetime DESC LIMIT 30;
