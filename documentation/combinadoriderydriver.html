7. 🔒 Consideraciones de Seguridad

⚠️ Importante: Las claves API mostradas en esta documentación son para propósitos de desarrollo. En producción, estas claves deben ser manejadas de forma segura.

7.1 Firebase Security

  • Firebase Auth: Autenticación de usuarios con tokens JWT
  • Firebase Rules: Reglas de seguridad para Firestore y Realtime Database
  • Crashlytics: Monitoreo de errores sin exponer datos sensibles

7.2 OneSignal Security

  • Rest Key: Usado para enviar notificaciones desde el servidor
  • App ID: Identificador público de la aplicación
  • Player IDs: Identificadores únicos de dispositivos

7.3 MQTT Security

  • Broker Público: HiveMQ es un broker público, considerar migración a broker privado
  • Topics: Usar topics específicos por usuario para evitar acceso no autorizado
  • SSL/TLS: Considerar implementar conexiones seguras

8. 🔧 Solución de Problemas

8.1 Problemas Comunes de Firebase

Problema Solución
Error de inicialización Verificar google-services.json y GoogleService-Info.plist
Error de autenticación Verificar configuración de OAuth en Firebase Console
Crashlytics no funciona Verificar que FlutterError.onError esté configurado

8.2 Problemas Comunes de OneSignal

Problema Solución
Notificaciones no llegan Verificar App ID y permisos de notificación
Error de Player ID Regenerar Player ID o reinstalar la app
Notificaciones en iOS Verificar configuración de certificados APNS

8.3 Problemas Comunes de MQTT

Problema Solución
No se conecta al broker Verificar conectividad de red y configuración del broker
Mensajes no llegan Verificar topics suscritos y QoS
Conexión inestable Implementar reconexión automática y keep-alive
