3.2 Conexión y Suscripción

mqttForUser() async { try { await client.connect(); } on NoConnectionException catch (e) { debugPrint(e.toString()); client.connect(); } if (client.connectionStatus!.state == MqttConnectionState.connected) { client.onSubscribed = onSubscribed; debugPrint('connected'); } else { client.connect(); } // Suscripción al topic específico del usuario client.subscribe('ride_request_status_' + sharedPref.getInt(USER_ID).toString(), MqttQos.atLeastOnce); }

3.3 Manejo de Mensajes

client.updates!.listen((List>? c) { final MqttPublishMessage recMess = c![0].payload as MqttPublishMessage; final pt = MqttPublishPayload.bytesToStringAsString(recMess.payload.message); if (jsonDecode(pt)['success_type'] == ACCEPTED || jsonDecode(pt)['success_type'] == ARRIVING || jsonDecode(pt)['success_type'] == ARRIVED || jsonDecode(pt)['success_type'] == IN_PROGRESS) { // Manejo de estados del viaje } else if (jsonDecode(pt)['success_type'] == CANCELED) { // Manejo de cancelación } else if (jsonDecode(pt)['success_type'] == COMPLETED) { // Manejo de completado } });

3.4 Estados del Viaje Manejados

Estado Descripción Acción
ACCEPTED Viaje aceptado por conductor Actualizar UI, mostrar conductor
ARRIVING Conductor en camino Mostrar ubicación conductor
ARRIVED Conductor llegó al destino Notificar llegada
IN_PROGRESS Viaje en progreso Iniciar seguimiento
COMPLETED Viaje completado Mostrar pantalla de reseña
CANCELED Viaje cancelado Volver al dashboard

5. Configuraciones de Seguridad

5.1 Firebase Security Rules

Importante: Las reglas de seguridad de Firebase deben configurarse adecuadamente para proteger los datos de los usuarios.

5.2 OneSignal Security

Rest Key: Utilizada para autenticación en las API de OneSignal
App ID: Identificador único de la aplicación
Player ID: Identificador único del dispositivo

5.3 MQTT Security

Nota: El broker HiveMQ público no requiere autenticación. Para producción, considerar un broker privado con autenticación.

7. Consideraciones de Producción

7.1 Escalabilidad

  • Firebase: Escalado automático según la demanda
  • OneSignal: Soporta millones de notificaciones por minuto
  • MQTT: Considerar broker privado para alta carga

7.2 Costos

  • Firebase: Plan gratuito con límites, escalado por uso
  • OneSignal: Plan gratuito hasta 10,000 suscriptores
  • MQTT: HiveMQ público gratuito, privado con costo

7.3 Mantenimiento

  • Monitoreo regular de logs y métricas
  • Actualización de dependencias
  • Backup de configuraciones
  • Testing de conectividad
